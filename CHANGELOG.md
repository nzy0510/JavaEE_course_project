# 更新日志

## 未发布

### 新增
- 文档型 RAG 知识库：新增文档上传、MarkItDown 转换、Markdown 清理切分、文档管理和切片查看能力。
- 新增 `knowledge_document` 与 `knowledge_chunk` 两表，分别保存文档元信息和可检索切片内容。
- AI 问答新增问题改写策略，模型可先生成 2-3 个检索查询，再按关键词和打分召回 top3 文档切片。
- 新增 MarkItDown 外部工具配置，支持通过 `MARKITDOWN_PYTHON` 指向本机 Python 虚拟环境。
- 新增文档知识分类识别，根据文件名关键词归入“大模型、Java基础、Spring、MySQL”等分类。
- 新增分类统计数据，统计接口返回各分类文档数和活跃切片数。
- 文档管理页保留文件名/状态查询、切片关键词查询，并新增文档永久删除操作。
- 新增 README，补充项目功能、启动运行、隐私配置、批量导入格式和验证说明。
- 登录页新增图片验证码，验证码写入 Session 并在登录校验后失效。
- 用户认证：支持用户名+密码注册、登录和退出，密码 BCrypt 加密存储，Session 会话管理。
- 数据可视化：首页改为知识库概览和最近文档，详细统计页保留题库分类及活动切片占比饼图、各知识分类切片数量柱形图。

### 变更
- 知识库运行模型从旧版 `knowledge_atom` 知识原子表改为文档/切片模型；旧 JSON 批量导入入口不再作为运行时知识库入口。
- `knowledge_document` 增加 `knowledge_category` 字段，AI 问答召回上下文同步展示片段所属知识分类。
- README、课程设计说明文档和页面文案同步改为文档 RAG 版本。
- 数据库密码和 DeepSeek API Key 改为通过环境变量读取，并提供 `.env.example` 作为本地配置示例。
- 本地默认启动配置恢复为 MySQL 直连方式，便于按课程项目环境直接运行。
- 打包方式调整为可执行 WAR，确保 JSP 页面资源随构建产物一起发布。
- 前端从 AI-Interview 项目的 Vue SPA 迁移为 JSP + Bootstrap 5 + jQuery，符合课程 JSP 架构要求。
- 移除 Qdrant 向量数据库、Redis 缓存、Flyway 数据库迁移、邮件服务等依赖。

### 修复
- 修复 JSP 页面脚本早于 jQuery 加载导致仪表盘、知识列表、添加和统计页面功能失效的问题。
- 修复未配置 AI Key 或模型调用异常时 AI 问答接口不可用的问题，现在会返回明确配置提示和已检索到的知识库内容。
- 缩短 AI 模型调用超时并减少重试次数，避免 DeepSeek 网络握手或响应异常时页面长时间卡住。
- 修复短问题标题无法成为切分边界的问题，提高 PDF 面试题类文档的切片稳定性。
